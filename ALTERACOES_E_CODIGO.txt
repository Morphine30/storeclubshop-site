 VISAO GERAL DA ARQUITETURA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  FRONTEND (storyclubshop-site/)
    Arquivos HTML, CSS e JS que o cliente ve no navegador.
    Hospedado no Netlify: https://storyclub.netlify.app
    Fala com o backend via fetch (chamadas HTTP).

  BACKEND (minha-api/)
    Servidor Node.js com Express.
    Hospedado no Render: https://minha-api-a6zk.onrender.com
    Conecta ao MongoDB Atlas (banco de dados).
    Conecta ao Cloudinary (armazenamento de imagens).
    Conecta ao MercadoPago (pagamentos).

  FLUXO DE UMA COMPRA:
    Navegador → script.js → API Express → MongoDB/Cloudinary/MercadoPago

  FLUXO DO ADMIN:
    admin.html → login com ADMIN_TOKEN → rotas protegidas da API

  FLUXO DO CLIENTE:
    conta.html → cadastro/login → JWT salvo no localStorage

 BACKEND — DESCRICAO DOS ARQUIVOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

[ app.js ]
  Configura o servidor Express com:
  - helmet(): adiciona headers de seguranca HTTP
  - rate limiting: max 100 req/15min geral, 20 req/15min em pagamentos
  - CORS com whitelist: so aceita requisicoes de localhost e FRONTEND_URL
  - Rotas: /api/produtos, /api/pagamento, /api/usuarios, /api/admin/verificar

[ server.js ]
  Ponto de entrada da aplicacao:
  - Carrega o .env com dotenv
  - Conecta ao MongoDB antes de ligar o servidor
  - Chama seed() para popular o banco se estiver vazio
  - Inicia o servidor na porta do .env (ou 3000)

[ db.js ]
  Conecta ao MongoDB Atlas via Mongoose usando a MONGODB_URI do .env.

[ cloudinaryConfig.js ]
  Configura o SDK do Cloudinary com as 3 credenciais do .env.

[ seed.js ]
  Insere os 12 produtos do produtos.json no banco se ele estiver vazio.
  Roda automaticamente na primeira inicializacao.

[ middlewares/auth.js ]
  Protege as rotas do admin (POST/PUT/DELETE de produtos).
  Verifica o header: Authorization: Bearer ADMIN_TOKEN
  Se o token estiver errado ou ausente, retorna erro 401.
  Se o ADMIN_TOKEN nao estiver configurado no .env, retorna erro 500.

[ middlewares/authUsuario.js ]
  Protege rotas de clientes logados (ex: ver perfil).
  Verifica o JWT no header Authorization.
  Se valido, adiciona req.usuario com os dados do cliente.
  Se invalido ou expirado, retorna erro 401.

[ models/Produto.js ]
  Schema do produto com os campos:
    id, nome, descricao, preco, precoAntigo, parcelas,
    estrelas, avaliacoes, badge, badgeOff, categoria,
    emoji, imagem, imagemPublicId, secao
  Configuracoes:
    - maxlength nos campos de texto (seguranca)
    - imagemPublicId e removido do JSON (nao expoe ao cliente)
    - _id e __v sao removidos do JSON

[ models/Usuario.js ]
  Schema do cliente com: nome, email, senha (hash bcrypt)
  - email e convertido para minusculo automaticamente
  - senha e removida do JSON (nao expoe ao cliente nunca)

[ controllers/produtosController.js ]
  CRUD completo:
  - listar(): retorna todos os produtos do MongoDB
  - obter(): retorna um produto por id numerico
  - criar(): recebe formulario, faz upload da imagem no Cloudinary, salva no banco
  - atualizar(): deleta imagem antiga do Cloudinary, faz upload da nova, atualiza banco
  - deletar(): deleta imagem do Cloudinary e remove do banco

[ controllers/pagamentoController.js ]
  - Recebe o carrinho do frontend (IDs e quantidades)
  - Busca os precos REAIS no MongoDB (nunca confia nos precos do cliente)
  - Cria a preferencia no MercadoPago com os itens validados
  - Retorna init_point (producao) e sandbox_init_point (teste)

[ controllers/usuariosController.js ]
  - registrar(): cria conta com senha criptografada (bcrypt, 12 rounds)
                 retorna JWT de 7 dias + dados do usuario
  - login(): verifica email e senha, retorna JWT + dados do usuario
  - perfil(): retorna dados do cliente logado (requer JWT valido)

[ routes/produtos.js ]
  GET    /api/produtos        → publico
  GET    /api/produtos/:id    → publico
  POST   /api/produtos        → protegido (auth admin)
  PUT    /api/produtos/:id    → protegido (auth admin)
  DELETE /api/produtos/:id    → protegido (auth admin)
  Upload de imagem via multer (memoryStorage, max 5MB, so JPG/PNG/WebP/GIF)

[ routes/usuarios.js ]
  POST /api/usuarios/registrar  → cria conta de cliente
  POST /api/usuarios/login      → faz login, retorna JWT
  GET  /api/usuarios/perfil     → retorna perfil (requer JWT)

[ routes/pagamento.js ]
  POST /api/pagamento/criar-preferencia → cria pagamento no MercadoPago

[ config.js ]
  Define window.API_URL automaticamente:
  - localhost/127.0.0.1 → http://localhost:3000
  - qualquer outro (Netlify) → https://minha-api-a6zk.onrender.com
  Incluido em index.html, admin.html e conta.html antes dos outros scripts.

[ index.html ]
  Loja principal. Alteracoes:
  - Botao "Conta" agora aponta para conta.html
  - Inclui config.js antes do script.js

[ script.js ]
  Logica completa da loja:
  - Carrega produtos da API ao abrir a pagina
  - Renderiza cards com imagem, preco, badge, etc
  - Carrinho completo (localStorage, adicionar, remover, alterar qtd)
  - Sidebar de carrinho (abre/fecha com animacao)
  - Checkout: POST para /api/pagamento → redireciona para MercadoPago
  - Toast de notificacao (verde/vermelho)
  - Scroll reveal, sticky header, busca, nav ativo
  - Escape de HTML (escHtml) para prevenir XSS

[ admin.html ]
  Painel do administrador. Recursos:
  - Tela de login com ADMIN_TOKEN (verifica contra /api/admin/verificar)
  - Token salvo em sessionStorage (some ao fechar o navegador)
  - Botao "Sair" no header
  - Lista todos os produtos com opcoes de editar e excluir
  - Formulario de cadastro/edicao com preview de imagem
  - Todas as chamadas mutantes incluem: Authorization: Bearer TOKEN
  - Respostas 401 forcam logout automatico

[ conta.html ]
  Pagina de conta do cliente. Recursos:
  - Abas: "Entrar" e "Criar Conta"
  - Cadastro: nome, email, senha (minimo 6 caracteres)
  - Login: email e senha
  - Token JWT salvo em localStorage (storyclub_token)
  - Dados do usuario salvos em localStorage (storyclub_usuario)
  - Exibe perfil (nome e email) quando logado
  - Botao "Sair" limpa localStorage e volta para o formulario
  - Separado do admin: nao da acesso ao painel

[ sucesso.html / erro.html / pendente.html ]
  Paginas exibidas apos o pagamento no MercadoPago:
  - sucesso.html: pagamento aprovado (verde)
  - erro.html: pagamento cancelado (vermelho)
  - pendente.html: pagamento em processamento (amarelo)
  Todas tem botao "Voltar a Loja".

[ netlify.toml ]
  - Define pasta raiz "." como diretorio de publicacao
  - Adiciona headers de seguranca: X-Frame-Options, X-Content-Type-Options,
    Referrer-Policy em todas as paginas

 VARIAVEIS DE AMBIENTE (.env)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  MONGODB_URI          string de conexao com o banco Atlas
  CLOUDINARY_CLOUD_NAME nome da conta no Cloudinary
  CLOUDINARY_API_KEY   chave publica da API Cloudinary
  CLOUDINARY_API_SECRET chave privada da API Cloudinary
  MP_ACCESS_TOKEN      token do MercadoPago (TEST- ou APP_USR-)
  FRONTEND_URL         URL do site Netlify (para liberar CORS)
  ADMIN_TOKEN          senha do painel admin (min 20 caracteres)
  JWT_SECRET           segredo para assinar tokens de clientes
  PORT                 porta do servidor (3000 local, auto no Render)

 DEPENDENCIAS INSTALADAS (npm)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  express        servidor HTTP
  cors           permite requisicoes entre dominios diferentes
  helmet         adiciona headers de seguranca HTTP
  express-rate-limit limita numero de requisicoes por IP
  multer         recebe upload de arquivos via formulario
  mongoose       ODM para MongoDB (schemas e queries)
  cloudinary     SDK para upload/delete de imagens
  mercadopago    SDK oficial para criar preferencias de pagamento
  dotenv         le o .env e injeta no process.env
  bcryptjs       criptografa e verifica senhas de clientes
  jsonwebtoken   gera e verifica tokens JWT de autenticacao
  nodemon        (dev) reinicia o servidor ao salvar arquivos

 SEGURANCA IMPLEMENTADA
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  - Helmet: headers HTTP que previnem XSS, clickjacking e MIME sniffing
  - Rate limiting: evita ataques de forca bruta e abuso da API
  - CORS com whitelist: so aceita requisicoes de origens conhecidas
  - ADMIN_TOKEN: rotas de escrita de produtos so aceitam requisicoes autenticadas
  - JWT: clientes autenticados com token de 7 dias assinado com segredo
  - bcrypt (12 rounds): senhas de clientes nunca salvas em texto puro
  - Precos do banco: pagamentoController busca precos no MongoDB,
    nunca confia nos precos enviados pelo cliente (previne manipulacao)
  - imagemPublicId oculto: ID interno do Cloudinary nao e exposto na API
  - Tipos de arquivo restritos: upload aceita so JPG, PNG, WebP e GIF (sem SVG)
  - escHtml(): todos os dados externos sao escapados antes de ir ao HTML
  - maxlength nos schemas: evita campos gigantes no banco
