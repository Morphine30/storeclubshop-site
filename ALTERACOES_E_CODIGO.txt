  O projeto tem duas partes separadas:

  FRONTEND (storeclubshop-site_backup/)
    Arquivos HTML, CSS e JS que o cliente ve no navegador.
    Hospedado no Netlify: https://storyclub.netlify.app
    Fala com o backend via fetch (chamadas HTTP).

  BACKEND (minha-api/)
    Servidor Node.js com Express.
    Hospedado no Render: https://minha-api-a6zk.onrender.com
    Conecta ao MongoDB (banco de dados na nuvem).
    Conecta ao Cloudinary (armazenamento de imagens).

  COMO FUNCIONA O FLUXO COMPLETO:
    Cliente → Netlify (HTML) → API Render → MongoDB/Cloudinary
    Admin   → admin.html → API Render (com autenticacao) → MongoDB/Cloudinary

┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/models/Produto.js                             │
└─────────────────────────────────────────────────────────────┘
  O que e:
    Define a "forma" de um produto no banco de dados MongoDB.

  Campos atuais:
    id              → numero unico (1, 2, 3...)
    nome            → nome do produto (obrigatorio)
    descricao       → descricao do produto (obrigatorio)
    preco           → preco atual em reais (obrigatorio)
    precoAntigo     → preco riscado (opcional)
    linkPagamento   → URL do link de compra por produto (opcional)
    estrelas        → avaliacao de 1 a 5
    avaliacoes      → numero de avaliacoes
    badge           → etiqueta colorida, ex: "🔥 Viral"
    badgeOff        → desconto, ex: "-35%"
    categoria       → Eletrônicos, Beleza, etc
    emoji           → emoji placeholder quando nao tem foto
    imagem          → URL da foto principal no Cloudinary
    imagemPublicId  → ID interno (para deletar do Cloudinary)
    imagens[]       → array com ate 4 fotos extras (URLs)
    imagensPublicIds[] → IDs internos das fotos extras
    secao           → "mais-vendidos" ou "novidades"

  Nota: imagemPublicId e imagensPublicIds nao aparecem nas
  respostas da API (removidos pelo toJSON transform).


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/models/Usuario.js                             │
└─────────────────────────────────────────────────────────────┘
  O que e:
    Define a conta de um cliente da loja.

  Campos:
    nome   → nome do cliente
    email  → e-mail (unico, em minusculo)
    senha  → senha criptografada com bcrypt (nao aparece na API)


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/controllers/produtosController.js             │
└─────────────────────────────────────────────────────────────┘
  O que e:
    O "cerebro" das operacoes de produtos.

  Funcoes:

    listar()
      Busca todos os produtos no MongoDB e retorna em JSON.

    obter()
      Busca um produto especifico pelo id numerico.

    criar()
      Recebe os dados do formulario (incluindo linkPagamento).
      Faz upload da imagem principal para o Cloudinary (se tiver).
      Faz upload de ate 4 fotos extras (se tiver).
      Salva tudo no MongoDB.

    atualizar()
      Usa findOneAndUpdate + $set para garantir que todos os
      campos (incluindo linkPagamento) sejam salvos corretamente.
      Se chegou nova imagem principal: deleta a antiga do Cloudinary
      e faz upload da nova.
      Se chegaram novas fotos extras: deleta todas as antigas
      do Cloudinary e faz upload das novas.

    deletar()
      Deleta a imagem principal do Cloudinary.
      Deleta todas as fotos extras do Cloudinary.
      Remove o produto do MongoDB.

  Nota tecnica importante:
    O metodo atualizar usa findOneAndUpdate com $set em vez de
    produto.save() para evitar um problema do Mongoose onde campos
    nulos nao eram detectados como modificados.


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/controllers/usuariosController.js             │
└─────────────────────────────────────────────────────────────┘
  O que e:
    Gerencia as contas dos clientes.

  Funcoes:
    registrar() → cria conta com senha criptografada (bcrypt 12 rounds)
                  retorna token JWT valido por 7 dias
    login()     → verifica senha com bcrypt, retorna JWT
    perfil()    → retorna dados do cliente (requer JWT valido)


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/routes/produtos.js                            │
└─────────────────────────────────────────────────────────────┘
  Endpoints da API de produtos:

    GET    /api/produtos         → lista todos (publico)
    GET    /api/produtos/:id     → retorna um produto (publico)
    POST   /api/produtos         → cria produto (requer ADMIN_TOKEN)
    PUT    /api/produtos/:id     → atualiza produto (requer ADMIN_TOKEN)
    DELETE /api/produtos/:id     → deleta produto (requer ADMIN_TOKEN)

  Upload de imagens:
    Usa upload.fields() do multer para aceitar:
      "imagem"   → 1 foto principal
      "imagens"  → ate 4 fotos extras
    Armazenamento em memoria (memoryStorage), envia para Cloudinary.
    Tamanho maximo por foto: 5MB
    Formatos aceitos: JPG, PNG, WEBP, GIF


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/routes/usuarios.js                            │
└─────────────────────────────────────────────────────────────┘
  Endpoints da API de clientes:

    POST /api/usuarios/registrar → cria conta de cliente
    POST /api/usuarios/login     → faz login, retorna JWT
    GET  /api/usuarios/perfil    → retorna dados (requer JWT)


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/middlewares/auth.js                           │
└─────────────────────────────────────────────────────────────┘
  O que e:
    Protege as rotas de admin (POST, PUT, DELETE de produtos).
    Exige o cabecalho: Authorization: Bearer ADMIN_TOKEN
    Se o token estiver errado, retorna erro 401 (nao autorizado).


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/middlewares/authUsuario.js                    │
└─────────────────────────────────────────────────────────────┘
  O que e:
    Protege a rota de perfil do cliente.
    Verifica o token JWT enviado pelo cliente logado.
    Anexa os dados do usuario em req.usuario.


┌─────────────────────────────────────────────────────────────┐
│ minha-api/scr/app.js                                        │
└─────────────────────────────────────────────────────────────┘
  Configuracoes de seguranca e rotas:

    helmet()        → cabecalhos de seguranca HTTP (XSS, clickjacking)
    cors()          → permite acesso apenas de https://storyclub.netlify.app
                      e localhost (para desenvolvimento)

┌─────────────────────────────────────────────────────────────┐
│ admin.html — Painel de administracao                        │
└─────────────────────────────────────────────────────────────┘
  O que e:
    Pagina de gerenciamento de produtos, protegida por senha.

  Funcionalidades:
    LOGIN:
      Overlay de senha ao abrir. So entra quem tem o ADMIN_TOKEN.
      Token salvo na sessao do navegador (sessionStorage).

    LISTA DE PRODUTOS (lado esquerdo):
      Mostra todos os produtos do banco com foto, nome, preco.
      Botao Editar: preenche o formulario com os dados do produto.
      Botao Lixeira: deleta com confirmacao.

    FORMULARIO (lado direito):
      Nome, Descricao, Preco, Preco Antigo
      Link de Pagamento (substitui o antigo campo "Parcelas")
      Categoria, Secao, Badge, Desconto, Estrelas, Avaliacoes, Emoji
      Imagem Principal: area clicavel com preview antes de salvar
      Fotos Extras: selecao de ate 4 fotos adicionais com preview
      Botao Salvar (adicionar) ou Atualizar (editar)
      Botao Cancelar aparece no modo de edicao

    AO EDITAR:
      Todos os campos sao preenchidos automaticamente.
      As fotos extras existentes aparecem como miniaturas.
      Enviar novas fotos extras substitui todas as antigas.
      Nao enviar novas fotos mantem as existentes.

 BUGS CORRIGIDOS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  1. linkPagamento nao salvava ao editar produto
     Causa: produto.save() do Mongoose nao detectava mudanca
            em campos que eram nulos desde sempre no banco.
     Solucao: trocado para findOneAndUpdate + $set que vai
              diretamente no MongoDB sem depender do rastreamento.

  2. Backend com codigo desatualizado no Render
     Causa: mudancas feitas localmente mas nao commitadas/enviadas.
     Solucao: sempre fazer git add + git commit + git push
              depois de qualquer mudanca no backend.

  3. Botao "Comprar Agora" cortado no mobile
     Causa: .product-card tinha overflow:hidden e o botao (<a>)
            nao tinha display:block.
     Solucao: removido overflow:hidden do card, adicionado
              display:block e white-space:nowrap no botao.

  4. Texto "Acordando o servidor" aparecia para o cliente
     Causa: mensagem tecnica exposta no overlay de carregamento.
     Solucao: trocado para mensagens neutras como "Buscando as
              melhores ofertas..." sem mencionar servidor.

 RESUMO DAS ROTAS DA API
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  GET    /                              API online (health check)
  GET    /api/produtos                  lista todos os produtos
  GET    /api/produtos/:id              retorna produto por id
  POST   /api/produtos                  cria produto [ADMIN_TOKEN]
  PUT    /api/produtos/:id              atualiza produto [ADMIN_TOKEN]
  DELETE /api/produtos/:id              deleta produto [ADMIN_TOKEN]
  GET    /api/admin/verificar           valida o ADMIN_TOKEN [ADMIN_TOKEN]

  POST   /api/usuarios/registrar        cria conta de cliente
  POST   /api/usuarios/login            faz login, retorna JWT
  GET    /api/usuarios/perfil           dados do cliente [JWT]

  POST   /api/pagamento/criar-preferencia   cria preferencia MercadoPago
